Opdracht 1: Dom (45 punten)

De sources voor deze opdracht vind je in Dom.zip. In de klasse Dom hoef je niets te wijzigen, tenzij je stukken code wilt uitschakelen met commentaar.

De uiteindelijk gewenste uitvoer:

node 'title' is already in tree

html
├─── head
│    ├─── title
│    └─── style
└─── body
     ├─── header
     ├─── main
     │    ├─── h1
     │    ├─── h2
     │    ├─── p
     │    ├─── h1
     │    ├─── h2
     │    └─── p
     └─── footer

tree depth = 4

De hoofdboom is hier het Element elemHtml.

Stap 1.1

  • Zorg dat een Element (zoals <title>) maar één keer in een boom kan voorkomen.
  • Maak daarvoor de methode isInTree in de klasse Element werkend. (15 punten)
  • Dit wordt (natuurlijk) een recursieve methode.
  • Wat is de efficiëntie in tijd en geheugengebruik van deze methode? Verklaar. (5 punten)

Stap 1.2

  • Zorg dat een FreeElement (zoals <h1> of <p>) wel vaker in een boom kan voorkomen.
  • Pas daarvoor alleen de subklasse FreeElement aan. (10 punten)

Stap 1.3

  • Schrijf de methode getDepth() om de diepte van een boom te bepalen. (10 punten)
  • Wat is de efficiëntie in tijd en geheugengebruik van deze methode? Verklaar. (5 punten)

Opdracht 2: Friends (55 punten)

De sources voor deze opdracht vind je in Friends.zip. In de klasse Friends hoef je niets te wijzigen, tenzij je stukken code wilt uitschakelen met commentaar.

De klasse FriendList bevat twee collecties:

  • friendsByName is een Map van vrienden, met als sleutel de naam van de vriend.
  • friendsByHobby is wat lastiger: een Map van Sets met als doel om per hobby een subset van de vrienden te kunnen opslaan

De uiteindelijk gewenste uitvoer:

My friends:
   Goofy Geef
   Michelle Dekker
   Joan Jet
   Marty McFly
   Rick Sanchez

Initaly:

My guitar friends:
   Joan Jet
   Marty McFly

My snowboarding friends:
   Goofy Geef
   Michelle Dekker

After updates:

My guitar friends:
   Joan Jet
   Marty McFly

My snowboarding friends:
   Goofy Geef
   Michelle Dekker

My travelling friends:
   Marty McFly
   Rick Sanchez

Stap 2.1

  • Initialiseer in de constructor van FriendList de twee Maps met geschikte implementaties. (10 punten)
  • Je keuze moet er voor zorgen dat de namen altijd alfabetisch gesorteerd worden weergegeven.
  • Schrijf in het commentaar de onderbouwing van je keuzes. (5 punten)
  • Afhankelijk van je keuzes, moet je wellicht iets toevoegen aan de klasse Friend (kijk goed naar eventuele foutmeldingen).

Stap 2.2

  • Schrijf de methode getByName in FriendList om een vriend op naam te kunnen zoeken. (10 punten)
  • Wat is de efficiëntie in tijd en geheugengebruik van deze methode? Verklaar. (5 punten)

Stap 2.3

  • Schrijf de methode addHobbyFriend in FriendList. Deze moet er voor zorgen dat een vriend aan het lijstje voor de betreffende hobby wordt toegevoegd. (15 punten)
  • In deze methode kijk je eerst of er al een lijst voor die hobby bestaat, en zo niet maak je die hier aan.
  • Aan de bestaande of net gemaakte lijst voeg je dan de vriend toe.

Stap 2.4

  • Als ik een nieuwe hobby krijg, wil ik de code van FriendList niet steeds aanpassen. Maak daarom de methode printByHobby algemener, zodat er geen hard-coded hobbynamen meer in voorkomen. (10 punten)
  • Als het goed is, worden de hobby's alfabetisch gerangschikt weergegeven en de namen daarbinnen ook.
